# UU Analytics — Руководство по ведению Google Таблицы

## Структура таблицы

Дашборд автоматически читает листы с названиями **«Свод»** и **«ДПО»**.  
Дополнительные листы добавляются в `config.js` → `window.EXTRA_SHEETS`.

---

## Лист «Свод»

### Обязательные колонки

| Колонка | Тип | Пример |
|---------|-----|--------|
| `Факультет` | Текст | `МШК`, `ФБМ`, `UU` |
| `Тип свода` | Текст | `Очное`, `Заочное`, `Общий` |
| `CSI Общий` | Число | `4.2` |
| *(любые метрики)* | Число | `3.8`, `4.5` |

### Правила заполнения

1. **Строка-benchmark** — строка, где `Факультет = UU`, отображается как пунктирная линия на графике (эталон по университету). Обязательна для каждого типа свода.
2. **Тип свода** — произвольный текст, появляется как кнопка-фильтр. Используйте единые названия (регистр важен).
3. **Метрики** — любые числовые колонки справа от идентификаторов. Дашборд определяет их автоматически. Дроби через `.` (точку), не запятую.
4. Пустые ячейки допустимы — строки с полностью пустыми значениями игнорируются.

### Пример

| Факультет | Тип свода | CSI Общий | NPS |
|-----------|-----------|-----------|-----|
| UU        | Общий     | 4.1       | 62  |
| МШК       | Общий     | 4.4       | 71  |
| ФБМ       | Общий     | 3.9       | 58  |
| UU        | Очное     | 4.3       | 65  |
| МШК       | Очное     | 4.6       | 74  |

## Лист «ДПО»

### Обязательные колонки

| Колонка | Тип | Пример |
|---------|-----|--------|
| `Школа` | Текст | `МШК`, `СКРИМ` |
| `Факультет` | Текст | `Маркетинг` |
| `Программа/курс` | Текст | `Digital Marketing Pro` |
| *(любые метрики)* | Число | `4.1` |

### Правила заполнения

1. Три первых колонки — идентификаторы, остальные числовые — метрики.
2. Фильтрация каскадная: Школа → Факультет → Программы/курсы.
3. Если школа выбрана в фильтре «Все школы», дашборд агрегирует по школам (среднее).
4. Названия школ в `SCHOOL_COLORS` (в `index.html`) определяют цвет столбца на графике. При добавлении новой школы рекомендуется добавить её цвет в словарь.

### Пример

| Школа | Факультет | Программа/курс | CSI Общий | Рекомендация |
|-------|-----------|----------------|-----------|--------------|
| МШК   | Маркетинг | Digital Pro    | 4.5       | 78           |
| МШК   | Маркетинг | Brand Manager  | 4.2       | 71           |
| СКРИМ | ИТ        | Python Base    | 3.9       | 65           |

## Дополнительные листы (generic)

Любой другой лист, добавленный в `EXTRA_SHEETS`, обрабатывается в «общем» режиме:

- Первая нечисловая колонка становится подписью оси.
- Все числовые колонки — метриками.
- Если есть ещё одна нечисловая колонка, она используется как группировка (фильтр «Группа»).

## Доступ к таблице

Таблица **должна быть открыта для просмотра по ссылке** (File → Share → Anyone with the link → Viewer).  
ID таблицы задаётся в `config.js`:

```js
window.SHEET_ID = '13sW5dzRGAMWEITuV-ivYKg7XS5TjOhncQC0KvmdLeJo';
```

ID — часть URL между `/d/` и `/edit`.

## Добавление новой метрики

1. Добавьте колонку с числовыми значениями на нужный лист.
2. Убедитесь, что заголовок понятен — он отображается в дропдауне «Метрика» и как заголовок графика.
3. Обновите дашборд кнопкой **Обновить** — новая метрика появится автоматически.

## Добавление нового типа свода / школы

1. Просто добавьте строки с новым значением в колонку `Тип свода` / `Школа`.
2. Для нового типа свода добавьте строку-benchmark (`Факультет = UU`) с нужным `Тип свода`.
3. Для новой школы в ДПО рекомендуется добавить цвет в `SCHOOL_COLORS` в `index.html`.

## KPI-карточки — что означают

| Карточка | Описание |
|----------|----------|
| **Среднее** | Среднее арифметическое всех отображаемых объектов |
| **Медиана** | Серединное значение — устойчиво к выбросам |
| **Стандартное откл.** | Мера разброса: чем меньше, тем однороднее показатели |
| **Максимум** | Наибольшее значение в выборке |
| **Минимум** | Наименьшее значение в выборке |
| **UU** | Benchmark строки `UU`; бейдж показывает отклонение среднего от него |
